# 定时策略接口需求文档

## 一、前端修改说明

### 1.1 策略类型调整

**原策略类型：**
- 工作日（weekday）
- 节假日（holiday）
- 每日（everyday）
- 时间段（timed）

**新策略类型：**
- 默认（default）- 合并原工作日、节假日、每日
- 时间段（timed）- 保持不变

### 1.2 默认策略特性

| 特性 | 说明 |
|------|------|
| 适用星期 | 需手动选择（周一至周日） |
| 时间冲突 | 同一星期不能同时被多个默认策略使用 |
| 优先级 | 低于时间段策略 |

### 1.3 时间段策略特性

| 特性 | 说明 |
|------|------|
| 执行日期 | 需指定开始日期和结束日期 |
| 适用星期 | 不显示，无需选择 |
| 优先级 | **最高**，可覆盖默认策略 |

### 1.4 前端校验逻辑

```javascript
// 默认策略提交时校验
function checkWeekdayConflict() {
  if (form.type !== 'default') return { valid: true }
  
  // 遍历其他启用的默认策略
  const otherStrategies = tableData.filter(s => 
    s.id !== currentId && 
    s.type === 'default' && 
    s.enabled !== false
  )
  
  // 检测星期冲突
  for (const other of otherStrategies) {
    const conflictDays = form.weekdays.filter(day => 
      other.weekdays.includes(day)
    )
    if (conflictDays.length > 0) {
      return { valid: false, conflictStrategy: other.name }
    }
  }
  return { valid: true }
}
```

## 二、后端接口需求

### 2.1 策略类型枚举

**数据库字段：`strategy.type`**

| 值 | 说明 | 前端显示 |
|----|------|----------|
| default | 默认策略 | 默认 |
| timed | 时间段策略 | 时间段 |

**迁移说明：**
- 原 `weekday`、`holiday`、`everyday` 类型的策略需迁移为 `default`
- 可在后端接口中做兼容处理：收到旧类型时统一映射为 `default`

### 2.2 策略实体字段

**表名：`timed_strategy`**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | bigint | 是 | 主键 |
| name | varchar(100) | 是 | 策略名称 |
| type | varchar(20) | 是 | 策略类型：default/timed |
| weekdays | json | 否 | 适用星期 [1,2,3,4,5,6,7]，默认类型必填 |
| start_date | date | 否 | 开始日期，时间段类型必填 |
| end_date | date | 否 | 结束日期，时间段类型必填 |
| start_time | time | 是 | 开始时间 |
| end_time | time | 是 | 结束时间 |
| brightness | int | 是 | 目标亮度 0-100 |
| district | varchar(50) | 否 | 行政区 |
| road | varchar(50) | 否 | 路段 |
| enabled | tinyint | 是 | 是否启用 0-禁用 1-启用 |
| create_time | datetime | 是 | 创建时间 |
| update_time | datetime | 是 | 更新时间 |

### 2.3 后端校验规则

#### 2.3.1 默认策略校验

```java
// 伪代码
if (strategy.getType().equals("default")) {
    // 1. 必须选择至少一个星期
    if (strategy.getWeekdays() == null || strategy.getWeekdays().isEmpty()) {
        throw new ValidationException("默认策略必须选择适用星期");
    }
    
    // 2. 校验星期冲突
    List<TimedStrategy> existing = strategyMapper.selectByTypeAndEnabled("default", true);
    for (TimedStrategy other : existing) {
        if (other.getId().equals(strategy.getId())) continue;
        
        Set<Integer> conflict = strategy.getWeekdays().stream()
            .filter(day -> other.getWeekdays().contains(day))
            .collect(Collectors.toSet());
            
        if (!conflict.isEmpty()) {
            throw new ConflictException(
                "星期冲突：" + other.getName() + "已占用" + conflict
            );
        }
    }
}
```

#### 2.3.2 时间段策略校验

```java
// 伪代码
if (strategy.getType().equals("timed")) {
    // 1. 必须指定日期范围
    if (strategy.getStartDate() == null || strategy.getEndDate() == null) {
        throw new ValidationException("时间段策略必须指定开始日期和结束日期");
    }
    
    // 2. 结束日期不能早于开始日期
    if (strategy.getEndDate().isBefore(strategy.getStartDate())) {
        throw new ValidationException("结束日期不能早于开始日期");
    }
}
```

### 2.4 策略执行优先级

**优先级规则（从高到低）：**

1. **时间段策略（timed）**
   - 当当前日期在策略的 start_date 和 end_date 范围内时生效
   - 可覆盖同一时间段内的默认策略

2. **默认策略（default）**
   - 根据 weekdays 判断当前星期是否匹配
   - 多个默认策略的星期不能重叠

**执行逻辑伪代码：**

```java
public Strategy getActiveStrategy(LocalDateTime now, String district, String road) {
    // 1. 先查找生效的时间段策略
    List<TimedStrategy> timedList = strategyMapper.selectTimedActive(
        now.toLocalDate(), district, road
    );
    if (!timedList.isEmpty()) {
        // 返回优先级最高的时间段策略（可按创建时间或ID排序）
        return timedList.get(0);
    }
    
    // 2. 再查找默认策略
    int weekday = now.getDayOfWeek().getValue(); // 1-7
    List<TimedStrategy> defaultList = strategyMapper.selectDefaultByWeekday(
        weekday, district, road
    );
    
    return defaultList.isEmpty() ? null : defaultList.get(0);
}
```

- **功能描述**: 分页查询定时策略列表

## 四、数据库迁移建议

### 4.1 类型字段迁移

```sql
-- 将旧类型统一更新为 default
UPDATE timed_strategy 
SET type = 'default' 
WHERE type IN ('weekday', 'holiday', 'everyday');

-- 如需保留原类型信息，可新增字段
ALTER TABLE timed_strategy ADD COLUMN old_type varchar(20) COMMENT '原策略类型';
UPDATE timed_strategy SET old_type = type WHERE type != 'default';
```

### 4.2 星期字段迁移

```sql
-- 原 everyday 类型的策略，weekdays 应设为 [1,2,3,4,5,6,7]
UPDATE timed_strategy 
SET weekdays = '[1,2,3,4,5,6,7]' 
WHERE old_type = 'everyday' AND (weekdays IS NULL OR weekdays = '[]');

-- 原 weekday 类型的策略，weekdays 应设为 [1,2,3,4,5]
UPDATE timed_strategy 
SET weekdays = '[1,2,3,4,5]' 
WHERE old_type = 'weekday' AND (weekdays IS NULL OR weekdays = '[]');

-- 原 holiday 类型的策略，weekdays 应设为 [6,7]
UPDATE timed_strategy 
SET weekdays = '[6,7]' 
WHERE old_type = 'holiday' AND (weekdays IS NULL OR weekdays = '[]');
```
